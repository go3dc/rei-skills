## 🛠️ Repository Philosophy
* **Modular Design:** Each skill is self-contained in its own directory with its own dependencies and documentation.
* **Resilience:** All skills prioritize idempotency (the ability to resume after failure) and observability.
* **Community-First:** Tools are provided "as-is" for educational purposes.

**Use these tools at your own risk.** * **Terms of Service:** Many real estate websites have Terms of Service (ToS) that prohibit automated scraping. You are responsible for ensuring your usage complies with the policies of the platforms you are querying.

## How to Contribute
If you would like to add a new skill to the library:
1. Create a new folder in the root directory (e.g., `/new-skill-name`).
2. Include the necessary source code, a `requirements.txt`, and a `README.md` for that specific tool.
3. Open a **Pull Request** to have it added to the index above.
